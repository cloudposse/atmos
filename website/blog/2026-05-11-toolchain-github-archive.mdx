Atmos's Aqua-compatible toolchain registry now understands the `github_archive` package type. Tools that ship as repository source archives — like `adr-tools` and `tfenv` — can now be installed through `atmos toolchain install` without any registry workarounds.

<!--truncate-->

## What Changed

Aqua's registry supports three package types for downloading tools:

- `github_release` — assets attached to a GitHub Release
- `http` — arbitrary HTTP(S) URLs
- `github_archive` — the auto-generated source tarball produced by GitHub for any tag

The first two were already supported in Atmos. `github_archive` was the gap, and any Aqua registry entry that used it would fail with `unsupported tool type: github_archive`. That gap is now closed.

The implementation matches upstream `aquaproj/aqua` exactly: the download URL resolves to `https://github.com/{owner}/{repo}/archive/refs/tags/{version}.tar.gz`, the archive is always `tar.gz` (per Aqua's `GetFormat()`), and the `asset`, `url`, and `format` fields are intentionally ignored for this type.

## How to Use It

For a tool that ships its binary as a script inside its source tree — such as `adr-tools`:

```yaml
packages:
  - type: github_archive
    repo_owner: npryce
    repo_name: adr-tools
    files:
      - name: adr
        src: adr-tools-{{trimV .Version}}/src/adr
```

The `{{trimV .Version}}` template expands to match GitHub's archive root directory (e.g., `adr-tools-3.0.0/` for version `v3.0.0`), so `files[].src` points to the file inside the extracted archive.

## Why This Matters

Aqua's upstream registry has hundreds of entries that use `github_archive` — `tfenv`, `tgswitch`, `adr-tools`, and many one-binary shell-script projects. Until now, those entries were dead in Atmos. Adding this single package type unblocks all of them without any registry-level changes from users: pull the Aqua registry entry as-is and it just works.
