You can now set **default arguments** for any Atmos command directly in `atmos.yaml`, addressed by
command path. Disable identity by default, turn off YAML-function processing for `describe`, or pin any
flag — and override it all with [profiles](/cli/configuration/profiles), environment variables, or on
the command line.

<!--truncate-->

## What Changed

Until now, every flag's default was hard-coded in Atmos. To change one you had to type it on every
invocation or export an environment variable.

Now you can list default args under a command's path in `atmos.yaml`:

```yaml
# global default args — applied to every command
args:
  - --identity=false

# per-command, path-derived
describe:
  args:
    - --process-functions=false
    - --skip=terraform.state
    - --skip=terraform.output

terraform:
  args:
    - --identity=false
```

`args` is the global list; `<command-path>.args` is per command (`describe.args`,
`describe.component.args`, `terraform.args`, …), with the most-specific path winning.

## Why Args Instead of a Typed Map

We deliberately chose raw argument tokens over a typed `flag: value` map. The map is tidy, but it's not
generic — it only covers flags Atmos models a specific way, and it needs new wiring as commands gain new
flags. Default **args** are the "plain stupid" option that's future-proof: any flag, any repeated flag,
even positionals, work with **zero per-command code**. It's exactly what you'd type, just remembered for
you.

This also means the mechanism is uniform — it works the same for `terraform`, `describe`, `list`,
`validate`, and everything else, because Atmos simply splices your default args onto the command line
before parsing.

## Precedence and Profiles

Default args are *defaults* — they never fight you:

```text
CLI flag  >  ENV var  >  command default args  >  global default args  >  built-in default
```

A default is skipped when you already pass that flag on the command line, or when its environment
variable is set. So `--process-functions=true` or `ATMOS_PROCESS_FUNCTIONS=true` always wins.

And because the args live in config, [profiles](/cli/configuration/profiles) override them for free —
a profile is merged into your configuration before args are resolved:

```yaml
# profiles/ci/settings.yaml — CI disables identity and YAML functions by default
args:
  - --identity=false
describe:
  args:
    - --process-functions=false
```

```shell
atmos describe component vpc -s prod --profile=ci
# or
ATMOS_PROFILE=ci atmos describe component vpc -s prod
```

## How to Use It

Add an `args:` list under the command's path (or top-level for global):

- Global → top-level `args:`
- `atmos describe ...` → `describe.args`
- `atmos describe component ...` → `describe.component.args`
- `atmos terraform ...` → `terraform.args`

Note the path is derived from the **command line**, so `terraform.args` is the command `atmos terraform`
— separate from `components.terraform`, which configures the terraform component type.
