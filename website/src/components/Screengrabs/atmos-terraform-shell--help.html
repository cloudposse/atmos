Usage: atmos terraform shell <component> -s <stack> The command does the following: - Processes the stack config files, generates the required variables for the Atmos component in the stack, and writes them to a file in the component's folder - Generates a backend config file for the Atmos component in the stack and writes it to a file in the component's folder (or as specified by the Atmos configuration setting) - Creates a Terraform workspace for the component in the stack - Drops the user into a separate shell (process) with all the required paths and ENV vars set - Inside the shell, the user can execute all Terraform commands using the native syntax For more details refer to https://atmos.tools/cli/commands/terraform/shell
