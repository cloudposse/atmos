Usage: atmos validate component [flags] Flags: -h, --help help for component --module-paths stringSlice Specify the paths to OPA policy modules or catalogs used for validating the component configuration in the given stack. (default [] ) --schema-path string Specify the path to the schema file used for validating the component configuration in the given stack, supporting schema types like jsonschema or opa. --schema-type string Validate the specified component configuration in the given stack using the provided schema file path and schema type ( jsonschema or opa ). -s, --stack string The stack flag specifies the environment or configuration set for deployment in Atmos CLI. --timeout int Validation timeout in seconds (default 0 ) Global Flags: --base-path string Base path for Atmos project --config stringSlice Paths to configuration files (comma-separated or repeated flag) (default [] ) --config-path stringSlice Paths to configuration directories (comma-separated or repeated flag) (default [] ) --logs-file string The file to write Atmos logs to. Logs can be written to any file or any standard file descriptor, including '/dev/stdout', '/dev/stderr' and '/dev/null' (default /dev/stderr ) --logs-level string Logs level. Supported log levels are Trace, Debug, Info, Warning, Off. If the log level is set to Off, Atmos will not log any messages (default Info ) --redirect-stderr string File descriptor to redirect stderr to. Errors can be redirected to any file or any standard file descriptor (including /dev/null ) Examples: • Specify the path to the schema file used for validating the component configuration in the given stack, supporting schema types like jsonschema or opa. $ atmos validate component <component> -s <stack> --schema-path <schema_path> --schema-type <jsonschema|opa> • Validate the specified component configuration in the given stack using the provided schema file path and schema type ( jsonschema or opa ). $ atmos validate component <component> -s <stack> --schema-path <schema_path> --schema-type <jsonschema|opa> • Specify the paths to OPA policy modules or catalogs used for validating the component configuration in the given stack. $ atmos validate component <component> -s <stack> --schema-path <schema_path> --schema-type opa --module-paths catalog • Specify validation timeout in seconds $ atmos validate component <component> -s <stack> --timeout 15 Use atmos validate component --help for more information about a command.
