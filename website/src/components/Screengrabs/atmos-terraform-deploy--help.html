Usage: atmos terraform deploy <component> -s <stack> <flags> The command automatically sets '-auto-approve' flag when running 'terraform apply'. It supports '--deploy-run-init=true|false' flag to enable/disable running terraform init before executing the command. It supports '--from-plan' flag. If the flag is specified, the command will use the planfile previously generated by 'atmos terraform plan' command instead of generating a new planfile. Note that in this case, the planfile name is in the format supported by Atmos and is saved to the component's folder. It supports '--planfile' flag to specify the path to a planfile. The '--planfile' flag should be used instead of the 'planfile' argument in the native 'terraform apply <planfile>' command. For more details refer to https://atmos.tools/cli/commands/terraform/deploy
