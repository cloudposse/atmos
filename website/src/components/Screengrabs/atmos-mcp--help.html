Atmos MCP provides two capabilities: MCP Server — Exposes Atmos AI tools to AI assistants (Claude Desktop, Claude Code, VSCode, Gemini CLI, Codex CLI) through the standardized MCP protocol. MCP Client — Connects to external MCP servers (AWS, GCP, Azure, custom) configured in atmos.yaml under mcp.servers. Their tools become available in atmos ai chat and atmos ai exec alongside native Atmos tools. Server commands: • atmos mcp start — Start the Atmos MCP server Client commands: • atmos mcp list — List configured external MCP servers • atmos mcp tools — List tools exposed by an MCP server • atmos mcp test — Test connectivity to an MCP server • atmos mcp status — Show status of all MCP servers • atmos mcp restart — Restart an MCP server • atmos mcp export — Export .mcp.json for IDE integration (Claude Code, Cursor) USAGE $ atmos mcp [sub-command] [flags] AVAILABLE COMMANDS export Export .mcp.json from atmos.yaml MCP server configuration list List configured MCP servers restart Validate that an MCP server can stop and restart cleanly (does not leave the server running) start Start Atmos MCP server status Show status of all MCP servers test Test connectivity to an MCP server tools List tools from an MCP server FLAGS -h, --help help for mcp GLOBAL FLAGS --ai Enable AI-powered analysis of command output --base-path string Base path for Atmos project -C, --chdir string Change working directory before executing the command (run as if Atmos started in this directory) --config strings Paths to configuration files (comma-separated or repeated flag) --config-path strings Paths to search for Atmos configuration (comma-separated or repeated flag) --force-color Force color output even when not a TTY (useful for screenshots) --force-tty Force TTY mode with sane defaults when terminal detection fails (useful for screenshots) --heatmap Show performance heatmap visualization after command execution (includes P95 latency) --heatmap-mode string Heatmap visualization mode: bar, sparkline, table (press 1-3 to switch in TUI) (default bar) --identity string Identity to use for authentication.
