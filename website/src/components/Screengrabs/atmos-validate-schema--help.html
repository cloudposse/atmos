Each entry under `schemas` should define: - `schema`: The path to the JSON schema file. - `matches`: A glob pattern that specifies which YAML files to validate. For every schema entry: - The JSON schema is loaded from the specified path. - All files matching the glob pattern are collected. - Each matching YAML file is parsed and converted to JSON. - The converted YAML is validated against the schema. This command helps ensure that configuration files follow a defined structure and are compliant with expected formats, reducing configuration drift and runtime errors.
