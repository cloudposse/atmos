<Intro>
When you import a file, your values override the imported values. That's the whole system.
</Intro>

## Imports

Every value you set in a file overrides the same value from any file you import. Later imports override earlier imports. This applies recursively at every level.

```yaml
import:
  - catalog/networking
  - catalog/compute
  - catalog/monitoring

vars:
  region: us-east-1  # Overrides region from any import above
```

## Scope Levels

Within each component, vars are merged from general to specific. More specific wins:

| Priority | Source | Example |
|----------|--------|---------|
| 1 (lowest) | Global vars | `vars:` |
| 2 | Component-type vars | `terraform.vars:` |
| 3 | Base component vars | From `metadata.inherits` |
| 4 | Component vars | `components.terraform.vpc.vars` |
| 5 (highest) | Override vars | `overrides.vars` ([file-scoped](/stacks/overrides)) |

Import order determines who wins **at the same scope level**. Scope level determines who wins **across levels**.

## The One Gotcha

If an imported file sets a value at the **component** level and you set it at the **global** level, the component level wins -- even though your file is doing the importing.

<File title="stacks/catalog/platform.yaml">
```yaml
components:
  terraform:
    vpc:
      vars:
        cidr_block: "10.0.0.0/16"   # Component level (priority 4)
```
</File>

<File title="stacks/deploy/prod-us.yaml">
```yaml
import:
  - catalog/platform

vars:
  cidr_block: "172.16.0.0/16"       # Global level (priority 1) — loses!
```
</File>

The fix: set it at the same level.

<File title="stacks/deploy/prod-us.yaml (fixed)">
```yaml
import:
  - catalog/platform

components:
  terraform:
    vpc:
      vars:
        cidr_block: "172.16.0.0/16"  # Same level — importing file wins
```
</File>

## Debugging

Use `atmos describe component` to see the final merged values:

```shell
atmos describe component vpc -s prod-us-east-1
```

Add `--provenance` to see exactly **where** each value came from — which file, line number, and whether it was defined inline, inherited, or overridden:

```shell
atmos describe component vpc -s prod-us-east-1 --provenance
```

Provenance shows symbol-based indicators for each value: `●` defined/overridden, `○` inherited, `∴` computed. Depth is shown as `[N]` — `[1]` for the parent stack, `[2+]` for deeper imports. See [`atmos describe component`](/cli/commands/describe/component) for details.
