## How Atmos Fits In

Atmos **replaces many single-purpose tools** while letting you bring in the tools you already know and love—like Terraform, OpenTofu, Helmfile, and Packer. Custom commands let you integrate everything else, and workflows simplify onboarding and bootstrapping.

| Capability | Atmos Approach |
|------------|----------------|
| **Infrastructure as Code** | Native Terraform, OpenTofu, Helmfile, Packer support |
| **Configuration** | Stack-based composition with imports and inheritance |
| **Task Automation** | Native [workflows](/workflows) with typed inputs |
| **Custom Commands** | Integrate any CLI tool with [custom commands](/cli/configuration/commands) |
| **Vendoring** | Native [vendoring](/vendor/) inspired by Vendir |
| **Policy** | Native [OPA validation](/cli/commands/validate/stacks) |
| **Dev Environments** | Native [devcontainer](/cli/commands/devcontainer) support |

## Design Inspirations

Atmos draws conceptual inspiration from several tools and frameworks that shaped its architecture:

### Component-Based Architecture (React)

React's component-based architecture inspired Atmos's approach to modularity. Just as React breaks UIs into reusable components, Atmos promotes breaking infrastructure into reusable components. Any Terraform "root module" can be used as a component, composed and configured through stacks.

### Template-Free Configuration (Kustomize)

Kustomize introduced a template-free way to customize Kubernetes configurations using overlays and inheritance. Atmos adopts a similar approach—enabling users to import, overlay, and override configurations without relying on templating. However, Atmos also supports advanced templating (via Gomplate) as an escape hatch when needed.

### Stack Orchestration (Helmfile)

Helmfile manages collections of Helm charts with declarative syntax, combining them into a "stack" for deployment. Atmos draws from Helmfile's ability to orchestrate multiple tools, applying the concept to Terraform root modules. Atmos generates `.tfvars` files like Helmfile generates Helm values files.

### Parameterized Packaging (Helm Charts)

Helm Charts provide a packaging format where parameterization enables flexibility. If your Terraform root modules are sufficiently parameterized, they function much like Helm charts—you only need to supply the appropriate values to achieve the desired configuration.

### Declarative Vendoring (Vendir)

Vendir by VMware Tanzu served as the basis for Atmos's vendoring implementation. After using Vendir, we realized we only needed a simpler subset of its functionality. We implemented our own version using HashiCorp's GoGetter library, adding OCI support for pulling configurations from anywhere.
