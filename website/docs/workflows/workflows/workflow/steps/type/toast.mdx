<Intro>
The `toast` step type displays a themed status message in success, info, warning, or error variants — the same status styling Atmos uses to report results across the CLI.
</Intro>

```yaml
steps:
  - name: done
    type: toast
    level: success
    content: Deployment complete.
```

## Fields

<dl>
  <dt>`content`</dt>
  <dd>Required. Message text. Supports templates.</dd>

  <dt>`level`</dt>
  <dd>Optional status level. Supported values include `success`, `info`, `warn`, `warning`, and `error`. The default is `info`.</dd>
</dl>

`success`, `info`, `warn`, and `error` are `level` values, not step types.
