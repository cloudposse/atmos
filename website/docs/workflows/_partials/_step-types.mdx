Atmos supports 25+ step types for workflows and custom command steps. Step type values live in the `type` field on a step object; type-specific fields stay on that same step object.

### Step Types Overview

| Category | Step Types | Description |
| --- | --- | --- |
| Command | [`atmos`](/workflows/steps/type/atmos), [`shell`](/workflows/steps/type/shell), [`exec`](/workflows/steps/type/exec), [`container`](/workflows/steps/type/container), [`http`](/workflows/steps/type/http) | Run Atmos, shell, process-replacement, or container operations, or call an HTTP endpoint |
| Interactive | [`input`](/workflows/steps/type/input), [`confirm`](/workflows/steps/type/confirm), [`choose`](/workflows/steps/type/choose), [`filter`](/workflows/steps/type/filter), [`file`](/workflows/steps/type/file), [`write`](/workflows/steps/type/write) | Collect user input in a TTY |
| UI | [`toast`](/workflows/steps/type/toast), [`markdown`](/workflows/steps/type/markdown), [`alert`](/workflows/steps/type/alert), [`say`](/workflows/steps/type/say), [`title`](/workflows/steps/type/title), [`clear`](/workflows/steps/type/clear), [`linebreak`](/workflows/steps/type/linebreak), [`stage`](/workflows/steps/type/stage), [`sleep`](/workflows/steps/type/sleep), [`env`](/workflows/steps/type/env), [`exit`](/workflows/steps/type/exit) | Display status, control terminal output, or influence workflow control flow |
| Output | [`spin`](/workflows/steps/type/spin), [`table`](/workflows/steps/type/table), [`pager`](/workflows/steps/type/pager), [`format`](/workflows/steps/type/format), [`join`](/workflows/steps/type/join), [`style`](/workflows/steps/type/style), [`log`](/workflows/steps/type/log) | Format, render, log, or capture output |

### Status Messages

Use `type: toast` for themed status messages. `success`, `info`, `warn`, and `error` are `level` values, not step types:

```yaml
steps:
  - name: done
    type: toast
    level: success
    content: Deployment complete.
```

### Interactive Steps

Interactive step types require a TTY. In CI, avoid prompts by using non-interactive command steps, defaults, or values supplied through environment variables.
