<Intro>
Use `atmos describe config` to view your fully-merged CLI configuration. This shows how Atmos is configured after loading and merging all configuration sources.
</Intro>

## View CLI Configuration

The `atmos describe config` command displays the complete, merged configuration that Atmos is using:

```bash
atmos describe config
```

<Screengrab title="atmos describe config" slug="atmos-describe-config" />

This command shows:
- Base paths for components, stacks, and workflows
- Stack naming patterns and included paths
- Terraform and Helmfile settings
- Terminal and logging configuration
- Active profiles and their overrides
- Integration settings (Atlantis, Spacelift, etc.)

## Output Formats

View configuration in different formats:

```bash
# YAML output (default)
atmos describe config

# JSON output
atmos describe config --format json

# Query specific values
atmos describe config --query '.stacks.base_path'
```

## Configuration Sources

The displayed configuration is merged from multiple sources in precedence order:

1. **Command-line flags** (highest priority)
2. **Environment variables** (`ATMOS_*`)
3. **Active profiles** (`--profile` or `ATMOS_PROFILE`)
4. **Configuration files** (`atmos.yaml`)
5. **Default values** (lowest priority)

## Debugging Configuration Issues

Use `atmos describe config` to troubleshoot:

```bash
# See where Atmos found its configuration file
atmos describe config --query '.cli_config_path'

# Check base paths
atmos describe config --query '.base_path'
atmos describe config --query '.stacks.base_path'
atmos describe config --query '.components.terraform.base_path'

# Verify profile settings are applied
ATMOS_PROFILE=developer atmos describe config
```
