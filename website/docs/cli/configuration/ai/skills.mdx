<Intro>
The `settings.ai.skills` section defines custom AI skills with specialized system prompts, tool permissions,
and behavior for different tasks.
</Intro>

<Experimental />

## Configuration

Set the default skill and optionally define custom skills in `atmos.yaml`. Marketplace skills (installed via `atmos ai skill install`) are available automatically.

<File title="atmos.yaml">
```yaml
settings:
  ai:
    default_skill: "atmos-stacks"    # Use a marketplace skill as default
    skills:
      # Custom skills are defined here (marketplace skills are loaded automatically)
      security-reviewer:
        display_name: "Security Reviewer"
        description: "Reviews infrastructure for security best practices"
        category: "security"
        system_prompt: |
          You are a security expert for Atmos infrastructure.
          Focus on access controls, encryption, and network security.
        allowed_tools:
          - atmos_describe_component
          - atmos_list_stacks
          - atmos_validate_stacks
          - read_stack_file
          - read_component_file
        restricted_tools: []
```
</File>

## Skill Schema

Each skill in the `skills` map supports the following settings:

<dl>
  <dt>`display_name`</dt>
  <dd>**Required.** Human-readable name shown in the skill selector (Ctrl+A) and session metadata.</dd>

  <dt>`description`</dt>
  <dd>**Required.** Brief description of what the skill does. Shown in the skill selector below the display name.</dd>

  <dt>`system_prompt`</dt>
  <dd>**Required.** System prompt that defines the AI's behavior, expertise, and response style for this skill. Use YAML multi-line (`|`) for longer prompts.</dd>

  <dt>`allowed_tools`</dt>
  <dd>List of tools available when this skill is active (default: `[]`). An empty list means all tools are available. Use specific tool names to restrict access.</dd>

  <dt>`restricted_tools`</dt>
  <dd>List of tools that require user confirmation before execution, even if they are in `allowed_tools` (default: `[]`).</dd>

  <dt>`category`</dt>
  <dd>Skill category for organization: `general`, `analysis`, `refactor`, `security`, `validation`, or `optimization`.</dd>
</dl>

## Global Skill Settings

<dl>
  <dt>`default_skill`</dt>
  <dd>Default skill to activate when starting a new chat session (default: `general`). Set at the `settings.ai` level.</dd>
</dl>

## Available Tools for Skills

<dl>
  <dt>Atmos Commands</dt>
  <dd>`atmos_describe_component`, `atmos_describe_affected`, `atmos_describe_stacks`, `atmos_describe_config`, `atmos_list_stacks`, `atmos_list_components`, `atmos_validate_stacks`, `atmos_get_template_context`</dd>

  <dt>File Operations</dt>
  <dd>`read_file`, `edit_file`, `write_file`, `read_stack_file`, `read_component_file`, `list_component_files`, `search_files`</dd>

  <dt>Execution</dt>
  <dd>`execute_bash`, `execute_atmos_command`</dd>

  <dt>Validation</dt>
  <dd>`validate_file_lsp` (if LSP is enabled)</dd>
</dl>

## Tool Safety Patterns

**Read-only custom skill** -- omit write and execute tools entirely:

<File title="atmos.yaml">
```yaml
skills:
  config-auditor:
    display_name: "Config Auditor"
    description: "Read-only configuration audit"
    system_prompt: "You audit stack configurations for issues..."
    allowed_tools:
      - atmos_describe_component
      - atmos_list_stacks
      - atmos_validate_stacks
      - read_file
      - search_files
    # No write/execute tools = read-only
```
</File>

**Write with confirmation** -- allow edits but require approval:

<File title="atmos.yaml">
```yaml
skills:
  component-refactorer:
    display_name: "Component Refactorer"
    description: "Refactors Terraform components"
    system_prompt: "You are an expert in Terraform component refactoring..."
    allowed_tools:
      - read_file
      - edit_file
      - list_component_files
      - execute_bash
    restricted_tools:
      - edit_file       # Require confirmation before editing
      - execute_bash    # Require confirmation before executing
```
</File>

## Marketplace Skills

Atmos includes 21 marketplace skills covering all aspects of infrastructure orchestration. Install them with:

```bash
atmos ai skill install cloudposse/atmos
```

<dl>
  <dt>`atmos-terraform`</dt>
  <dd>Terraform orchestration: plan/apply/deploy, workspace management, backend config, varfile generation, authentication.</dd>

  <dt>`atmos-stacks`</dt>
  <dd>Stack configuration: imports, inheritance, deep merging, locals, vars, settings, metadata, overrides.</dd>

  <dt>`atmos-components`</dt>
  <dd>Component architecture: Terraform root modules, abstract components, inheritance, versioning, mixins, catalog patterns.</dd>

  <dt>`atmos-validation`</dt>
  <dd>Policy validation: OPA/Rego policies, JSON Schema, schema manifests.</dd>

  <dt>`atmos-config`</dt>
  <dd>Project configuration: atmos.yaml structure, all sections, discovery, merging, base paths, settings.</dd>

  <dt>`atmos-auth`</dt>
  <dd>Authentication and identity management: providers (SSO/SAML/OIDC/GCP), identities, keyring, identity chaining.</dd>

  <dt>`atmos-templates`</dt>
  <dd>Go templates: Sprig/Gomplate functions, atmos.Component, atmos.GomplateDatasource, atmos.Store.</dd>

  <dt>`atmos-yaml-functions`</dt>
  <dd>YAML functions: !terraform.state, !terraform.output, !store, !env, !exec, !include, !template.</dd>

  <dt>`atmos-vendoring`</dt>
  <dd>Component vendoring: vendor.yaml manifests, pulling from Git/S3/HTTP/OCI/Terraform Registry.</dd>

  <dt>`atmos-workflows`</dt>
  <dd>Workflow automation: multi-step workflows, Go template support, cross-component orchestration.</dd>

  <dt>`atmos-gitops`</dt>
  <dd>CI/CD: GitHub Actions, Spacelift, Atlantis, `atmos describe affected` for change detection.</dd>

  <dt>`atmos-design-patterns`</dt>
  <dd>Design patterns: stack organization, component catalogs, inheritance, configuration composition.</dd>
</dl>

See [AI Skills](/ai/skills) for the full list and usage details.

## Custom Skill Examples

Custom skills complement marketplace skills for organization-specific needs.

### Security Reviewer

<File title="atmos.yaml">
```yaml
settings:
  ai:
    skills:
      security-reviewer:
        display_name: "Security Reviewer"
        description: "Reviews infrastructure for security best practices"
        category: "security"
        system_prompt: |
          You are a security expert reviewing Atmos infrastructure.

          FOCUS AREAS:
          - Network security (security groups, NACLs, VPC design)
          - Encryption (at rest and in transit)
          - IAM policies and least-privilege access
          - Compliance requirements (SOC2, HIPAA)

          Always use tools to gather actual config before making recommendations.
        allowed_tools:
          - atmos_describe_component
          - atmos_list_stacks
          - atmos_validate_stacks
          - read_stack_file
          - read_component_file
          - read_file
          - search_files
```
</File>

### Migration Assistant

<File title="atmos.yaml">
```yaml
settings:
  ai:
    skills:
      migration-assistant:
        display_name: "Migration Assistant"
        description: "Assists with infrastructure migrations and upgrades"
        category: "refactor"
        system_prompt: |
          You are an infrastructure migration expert.

          MIGRATION FOCUS:
          - Upgrade deprecated Terraform syntax
          - Update provider version constraints
          - Modernize component configurations
          - Ensure backward compatibility

          Make incremental changes and verify at each step.
        allowed_tools:
          - read_file
          - edit_file
          - list_component_files
          - read_component_file
          - search_files
          - execute_bash
        restricted_tools:
          - edit_file       # Require confirmation before editing
          - execute_bash    # Require confirmation before executing
```
</File>
