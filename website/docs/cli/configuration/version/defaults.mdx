`version.defaults` sets shared policy for managed versions. Track defaults, entry fields, and group policies can override these values.

## Configuration

<File title="atmos.yaml">
```yaml
version:
  defaults:
    update:
      strategy: patch
      cooldown: 14d
    include:
      - "v1.*"
    exclude:
      - "*-rc*"
      - "*-beta*"
    prerelease: false
    labels:
      - dependencies
```
</File>

## Fields

<dl>
  <dt>`update.strategy`</dt>
  <dd>Maximum update scope: `major`, `minor`, `patch`, or `pin`.</dd>

  <dt>`update.cooldown`</dt>
  <dd>Minimum release age before an update is eligible, such as `14d`, `2w`, or `36h`.</dd>

  <dt>`update.pin`</dt>
  <dd>Artifact form to render. Use `digest` or `sha` for immutable identifiers, or `none` for plain versions.</dd>

  <dt>`include`</dt>
  <dd>Candidate version patterns to allow. When set, versions must match at least one pattern.</dd>

  <dt>`exclude`</dt>
  <dd>Candidate version patterns to block. Excluded versions cannot be selected, even when requested directly.</dd>

  <dt>`prerelease`</dt>
  <dd>Whether prerelease candidates are eligible. Defaults to `false`.</dd>

  <dt>`labels`</dt>
  <dd>Metadata labels for downstream workflows and reports.</dd>
</dl>

## Example

```yaml
version:
  defaults:
    update:
      strategy: minor
      cooldown: 7d
    exclude:
      - "*-alpha*"
    prerelease: false
```
