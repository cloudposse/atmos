<Intro>
The Atmos MCP server uses the `ai` section of your `atmos.yaml` to determine which tools are available and how
they behave. The MCP server itself requires no additional configuration beyond enabling AI tools.
</Intro>

<Experimental />

## Quick Start

<File title="atmos.yaml">
```yaml
ai:
  enabled: true
  tools:
    enabled: true
```
</File>

## Configuration Reference

### Minimal Setup

The only required configuration is enabling AI tools in `atmos.yaml`:

<File title="atmos.yaml">
```yaml
ai:
  enabled: true
  tools:
    enabled: true
```
</File>

### Full Configuration

For more control, you can specify providers, tool restrictions, and confirmation behavior:

<File title="atmos.yaml">
```yaml
ai:
  enabled: true
  default_provider: "anthropic"
  providers:
    anthropic:
      model: "claude-sonnet-4-6"
      api_key: !env "ANTHROPIC_API_KEY"

  tools:
    enabled: true
    require_confirmation: true
    allowed_tools:
      - atmos_describe_component
      - atmos_list_stacks
      - atmos_validate_stacks
    blocked_tools:
      - execute_bash_command
```
</File>

:::note
The MCP server itself does not require an API key. The tools it exposes use your configured AI provider settings.
Provider configuration is only needed if you also use `atmos ai chat` or `atmos ai ask`.
:::

### Tool Settings

The MCP server exposes all tools configured in `ai.tools`. See [AI Tools Configuration](/cli/configuration/ai/tools) for the full reference of tool settings including:

<dl>
  <dt>`tools.enabled`</dt>
  <dd>Enable or disable tool execution (default: `false`). Must be `true` for the MCP server to expose any tools.</dd>

  <dt>`tools.require_confirmation`</dt>
  <dd>When accessed via MCP, tools execute without interactive confirmation prompts. Use `allowed_tools` and `blocked_tools` to control access.</dd>

  <dt>`tools.allowed_tools`</dt>
  <dd>Tools available for execution. Supports wildcards like `atmos_describe_*` (default: `[]` = all tools).</dd>

  <dt>`tools.blocked_tools`</dt>
  <dd>Tools completely blocked from execution (default: `[]`).</dd>
</dl>

### Available Tools

When connected, the MCP server exposes all enabled Atmos AI tools:

<dl>
  <dt>`atmos_describe_component`</dt>
  <dd>Get detailed component configuration for a stack.</dd>

  <dt>`atmos_list_stacks`</dt>
  <dd>List and filter stacks.</dd>

  <dt>`atmos_validate_stacks`</dt>
  <dd>Validate stack configurations.</dd>

  <dt>`describe_affected`</dt>
  <dd>Show components affected by git changes.</dd>

  <dt>`read_component_file`</dt>
  <dd>Read a component source file.</dd>

  <dt>`read_stack_file`</dt>
  <dd>Read a stack configuration file.</dd>

  <dt>`write_component_file`</dt>
  <dd>Modify a component file.</dd>

  <dt>`write_stack_file`</dt>
  <dd>Modify a stack file.</dd>
</dl>

All tools respect your configured [permission settings](/cli/configuration/ai/tools).
