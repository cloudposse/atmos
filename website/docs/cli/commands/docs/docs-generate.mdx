:::note Purpose

Use this command to generate a README.md from one or more input sources.  

<Screengrab title="atmos docs generate readme --help" slug="atmos-docs-generate-readme--help" />

The sources are defined at `docs.generate.readme` section of `atmos.yaml`.

    ```yaml
    docs:
    generate:
      # Generate README in current working directory
      readme:
        base-dir: .
        input:
          - "./README.yaml"
        # A template can be a remote URL/github/gitlab/etc 
        template:  "https://raw.githubusercontent.com/cloudposse/.github/5a599e3b929f871f333cb9681a721d26b237d8de/README.md.gotmpl"
        # The final README
        output: "README.md"
        terraform:
          source: src/
          enabled: false
          format: "markdown"
          show_providers: false
          show_inputs: true
          show_outputs: true
          sort_by: "name"
          hide_empty: false
          indent_level: 2

    ```

The command combines all local or remote YAML files specified at `input` and template file then generates README.md at the `output` folder. In case the template contains `terraform_docs` key, e.g.  

    ```yaml
{{- $data := (ds "config") -}}

{{ $data.name | default "Project Title" }}

{{ $data.description | default "No description provided." }}

{{ if has $data "extra_info" }}
Extra info: {{ $data.extra_info }}
{{ end }}

{{ if has $data "terraform_docs" }}
## Terraform Docs
{{ $data.terraform_docs }}
{{ end }}

    ```
the resultant README.md will also have a corresponding section rendered. By default `terraform.format` is set to `markdown table`, and can also be `markdown`, `tfvars hcl`, and `tfvars json`.  
This command also aliases to “atmos generate docs”.

:::

## Usage

```shell
atmos docs generate readme

``` 
## Supported Sources for README.yaml and template

### Local Sources

The `!include` function supports the following local file sources:
  - Absolute paths
    ```yaml
    input: - /Users/me/Documents/README.yaml
    template: /Users/me/Documents/README.md.gotmpl
    ```

  - Paths relative to the current working directory
    ```yaml
    input: - ./README.yaml
    template: ./README.md.gotmpl
    ```

  - Paths relative to the `base_dir` defined in `atmos.yaml` CLI config file (then resolved as relative to cwd)
    ```yaml
    input: - terraform/README.yaml
    template: terraform/README.md.gotmpl
    ```

### Remote Sources

To download remote files, Atmos uses [`go-getter`](https://github.com/hashicorp/go-getter)
(used by [Terraform](https://www.terraform.io/) for downloading modules)
