<Intro>
Use this command to manage the AI project memory file (ATMOS.md), which stores project-specific context, patterns, and conventions that help the AI provide more accurate and relevant assistance.
</Intro>

## Description

The `atmos ai memory` command provides tools to initialize, view, validate, edit, and locate your project memory file. Project memory allows the AI assistant to remember your organization's specific patterns, common commands, stack structures, and infrastructure conventions.

## Usage

```shell
atmos ai memory [subcommand]
```

## Subcommands

<dl>
  <dt>`init`</dt>
  <dd>Create a new ATMOS.md template file in your project</dd>

  <dt>`show`</dt>
  <dd>Display the current project memory content</dd>

  <dt>`validate`</dt>
  <dd>Validate the ATMOS.md file format and structure</dd>

  <dt>`edit`</dt>
  <dd>Open the ATMOS.md file in your editor ($EDITOR)</dd>

  <dt>`path`</dt>
  <dd>Show the absolute path to the ATMOS.md file</dd>
</dl>

## Configuration

Enable project memory in your `atmos.yaml`:

```yaml
settings:
  ai:
    enabled: true
    provider: "anthropic"
    memory:
      enabled: true                # Enable project memory
      file_path: "ATMOS.md"       # Memory file location (default)
      create_if_missing: true      # Auto-create template if needed
      auto_update: false           # Manual updates only (recommended)
      sections:                    # Sections to include in AI context
        - project_context
        - common_commands
        - stack_patterns
        - frequent_issues
        - infrastructure_patterns
        - component_catalog
        - team_conventions
        - recent_learnings
```

  ai:

  <dd>Overwrite existing ATMOS.md file</dd>

### atmos ai memory edit

Open the ATMOS.md file in your preferred text editor.

<Terminal title="atmos ai memory edit">
```console
$ atmos ai memory edit

Opening ATMOS.md in vim...

✅ Memory file saved
```
</Terminal>

Uses the editor specified in `$EDITOR` environment variable, or `vim` if not set.

**Setting your editor:**

```shell
export EDITOR="code"    # VS Code
export EDITOR="nano"    # Nano
export EDITOR="emacs"   # Emacs
```

If the ATMOS.md file doesn't exist, it will be created automatically with the default template.

- Locating the file quickly

## Memory File Structure

The ATMOS.md file is a standard Markdown document with level 2 headers (`##`) defining sections:

```markdown
# Atmos Project Memory

> This file is automatically maintained by Atmos AI to remember project-specific
> context, patterns, and conventions. Edit freely - AI will preserve manual changes.

## Project Context

**Organization:** your-org
**Atmos Version:** 1.89.0
**Primary Regions:** us-east-1, us-west-2
**Environments:** dev, staging, prod

## Common Commands

\```bash
# Describe a component
atmos describe component <component> -s <stack>
\```

## Stack Patterns

### Stack Structure
\```yaml
# Common import pattern
import:
  - catalog/stacks/baseline
  - catalog/stacks/network
\```

## Frequent Issues & Solutions

### Stack not found error
**Problem:** `Error: stack 'my-stack' not found`
**Solution:** Check stack naming and verify config exists in stacks/ directory

## Infrastructure Patterns

### Multi-Region Setup
- Primary region for production workloads
- DR region with replication
- Cross-region networking configured

## Component Catalog Structure

\```
components/
  terraform/
    vpc/           # VPC and networking
    rds/           # RDS databases
    eks/           # EKS clusters
\```

## Team Conventions

- All infrastructure changes require PR review
- Use `atmos validate` before committing
- Tag all resources appropriately

## Recent Learnings

*AI will add notes here as it learns about the project*
```

## Examples

### Initialize and Configure

<Terminal title="Setting up project memory">
```console
$ # Initialize memory file
$ atmos ai memory init

$ # Edit to customize for your project
$ atmos ai memory edit

$ # Validate the format
$ atmos ai memory validate

$ # View what will be sent to AI
$ atmos ai memory show
```
</Terminal>

### View Memory Path

```shell
# Find memory file location
atmos ai memory path

# Open in your favorite editor
code $(atmos ai memory path)

# Add to version control
git add $(atmos ai memory path)
git commit -m "Add project memory for AI assistant"
```

### Validation Workflow

```shell
# After editing, validate format
atmos ai memory validate

# If valid, commit changes
if atmos ai memory validate; then
  git add ATMOS.md
  git commit -m "Update project memory"
fi
```

## Troubleshooting

### Memory not being used

1. Check that memory is enabled:
```shell
grep -A 5 "memory:" atmos.yaml
```

2. Verify file exists:
```shell
atmos ai memory path
ls -la $(atmos ai memory path)
```

3. Validate file format:
```shell
atmos ai memory validate
```

### File not found

Create the file:
```shell
atmos ai memory init
```

Or enable auto-creation in `atmos.yaml`:
```yaml
ai:
  memory:
    create_if_missing: true
```

### Validation errors

Check that:
- File is valid UTF-8 text
- Sections use level 2 headers (`##`)
- No special characters in section names
