## Description

Project instructions allow the AI assistant to use your organization's specific patterns, common commands, stack structures, and infrastructure conventions. The instructions are stored in an `ATMOS.md` file in your project root.

## Configuration

Enable project instructions in your `atmos.yaml`:

```yaml
settings:
  ai:
    enabled: true
    provider: "anthropic"
    memory:
      enabled: true                # Enable project instructions
      file: "ATMOS.md"            # Instructions file location (default)
      create_if_missing: true      # Auto-create template if needed
      auto_update: false           # Manual updates only (recommended)
      sections:                    # Sections to include in AI context
        - project_context
        - common_commands
        - stack_patterns
        - frequent_issues
        - infrastructure_patterns
        - component_catalog
        - team_conventions
        - recent_learnings
```

## Editing the Instructions File

Edit `ATMOS.md` with any text editor:

<Terminal title="shell">
```bash
$ code ATMOS.md           # VS Code
$ vim ATMOS.md            # Vim
$ nano ATMOS.md           # Nano
```
</Terminal>

Changes take effect immediately on the next AI request.

## Instructions File Structure

The ATMOS.md file is a standard Markdown document with level 2 headers (`##`) defining sections:

```markdown
# Atmos Project Instructions

> This file provides project-specific context, patterns, and conventions to the
> Atmos AI Assistant. Edit freely - AI will preserve manual changes.

## Project Context

**Organization:** your-org
**Atmos Version:** 1.89.0
**Primary Regions:** us-east-1, us-west-2
**Environments:** dev, staging, prod

## Common Commands

\```bash
# Describe a component
atmos describe component <component> -s <stack>
\```

## Stack Patterns

### Stack Structure
\```yaml
# Common import pattern
import:
  - catalog/stacks/baseline
  - catalog/stacks/network
\```

## Frequent Issues & Solutions

### Stack not found error
**Problem:** `Error: stack 'my-stack' not found`
**Solution:** Check stack naming and verify config exists in stacks/ directory

## Infrastructure Patterns

### Multi-Region Setup
- Primary region for production workloads
- DR region with replication
- Cross-region networking configured

## Component Catalog Structure

\```
components/
  terraform/
    vpc/           # VPC and networking
    rds/           # RDS databases
    eks/           # EKS clusters
\```

## Team Conventions

- All infrastructure changes require PR review
- Use `atmos validate` before committing
- Tag all resources appropriately

## Recent Learnings

*AI will add notes here as it learns about the project*
```

2. **Be Concise**: Focus on frequently used patterns and commands
3. **Use Examples**: Include real command examples from your project
4. **Version Control**: Commit ATMOS.md to Git
5. **Team Review**: Have team review and approve changes
6. **Optimize Size**: Keep file under 10KB for optimal performance

## Troubleshooting

### Instructions not being used

1. Check that instructions are enabled:
```shell
grep -A 5 "memory:" atmos.yaml
```

2. Verify file exists:
```shell
ls -la ATMOS.md
```

### File not found

Create the file manually or enable auto-creation in `atmos.yaml`:
```yaml
ai:
  memory:
    create_if_missing: true
```

### Validation errors

Check that:
- File is valid UTF-8 text
- Sections use level 2 headers (`##`)
- No special characters in section names
