## Description

The `atmos ai help` command provides in-depth, context-aware explanations of Atmos concepts and features. Unlike general questions with `atmos ai ask`, this command is optimized for learning and understanding specific topics.

The AI provides:
- Detailed concept explanations
- Best practices and recommendations
- Code examples and patterns
- Common pitfalls and how to avoid them
- Links to related topics

## Usage

```shell
atmos ai help [topic]
```

If no topic is provided, the AI provides a comprehensive overview of Atmos.

<dl>

## Configuration

Configure an AI provider in your `atmos.yaml`:

```yaml
settings:
  ai:
    enabled: true
    provider: "anthropic"
    api_key_env: "ANTHROPIC_API_KEY"
```

## Output Format

The help command provides:

1. **Preparation indicator** - Shows content is being prepared
2. **Help header** - Indicates the topic being explained
3. **Structured content** - Well-organized sections with:
   - Concept definitions
   - Code examples
   - Best practices
   - Common patterns
   - Related topics

## Advanced Usage

### Learning Paths

```shell
# New user learning path
atmos ai help general          # Start with overview
atmos ai help stacks          # Learn about stacks
atmos ai help components      # Understand components
atmos ai help workflows       # Master workflows

# Advanced user topics
atmos ai help templating
atmos ai help validation
atmos ai help vendoring
```

### Provider-Specific Help

Different AI providers may offer varying levels of detail:

```shell
# Claude (Anthropic) - detailed, structured explanations
export ANTHROPIC_API_KEY="your-key"
atmos ai help stacks

# GPT (OpenAI) - practical examples and patterns
export OPENAI_API_KEY="your-key"
atmos ai help workflows

# Gemini (Google) - comprehensive, fast responses
export GEMINI_API_KEY="your-key"
atmos ai help templating
```
