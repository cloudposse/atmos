## Description

The `atmos ai security` command connects to AWS security services via Atmos Auth, retrieves security findings, and maps them to the Terraform/Atmos components that manage the affected resources. The AI analyzes each finding and generates remediation guidance with concrete code changes you can apply to your infrastructure.

Use it for:
- **Security Posture Review**: Get a prioritized view of security findings mapped to your Atmos components and stacks
- **Remediation Planning**: Receive AI-generated code changes to fix security issues in your Terraform components
- **CI/CD Integration**: Export findings as JSON or CSV for automated security gates in deployment pipelines
- **Compliance Reporting**: Filter findings by severity, source, or compliance framework for targeted reports

## Usage

```shell
atmos ai security [flags]
```

## Flags

<dl>
  <dt>`--stack, -s`</dt>
  <dd>Filter findings to a specific Atmos stack (e.g., `prod-us-east-1`).</dd>

  <dt>`--component, -c`</dt>
  <dd>Filter findings to a specific Atmos component (e.g., `vpc`, `eks`).</dd>

  <dt>`--severity`</dt>
  <dd>Comma-separated list of severity levels to include: `critical`, `high`, `medium`, `low`, `informational`.</dd>

  <dt>`--source`</dt>
  <dd>Filter findings by source service: `security-hub`, `config`, `inspector`, `guardduty`.</dd>

  <dt>`--format, -f`</dt>
  <dd>Output format: `text`, `json`, `csv`, `markdown` (default: `text`).</dd>

  <dt>`--max-findings`</dt>
  <dd>Maximum number of findings to retrieve and analyze.</dd>

  <dt>`--no-ai`</dt>
  <dd>Output raw findings without AI analysis or remediation suggestions.</dd>

  <dt>`--region`</dt>
  <dd>AWS region to query for findings (overrides stack region).</dd>

  <dt>`--framework`</dt>
  <dd>Filter findings by compliance framework (e.g., `cis-aws`, `pci-dss`).</dd>
</dl>

## Examples

### Basic Usage

<Terminal title="basic examples">
```bash
# Analyze findings for a specific stack
atmos ai security --stack prod-us-east-1

# Filter by severity
atmos ai security --stack prod-us-east-1 --severity critical,high

# Filter by source service
atmos ai security --stack prod-us-east-1 --source security-hub
```
</Terminal>

### Output Formats

<Terminal title="output format examples">
```bash
# Output as JSON for CI/CD integration
atmos ai security --stack prod-us-east-1 --format json

# Output as CSV for compliance reporting
atmos ai security --format csv > findings.csv

# Markdown report for documentation
atmos ai security --stack prod-us-east-1 --format markdown --severity critical,high
```
</Terminal>

### Filtering and Targeting

<Terminal title="filtering examples">
```bash
# Findings for a specific component
atmos ai security --stack prod-us-east-1 --component vpc

# Raw findings without AI analysis
atmos ai security --stack prod-us-east-1 --no-ai

# Limit number of findings
atmos ai security --stack prod-us-east-1 --max-findings 20

# Filter by compliance framework
atmos ai security --stack prod-us-east-1 --framework cis-aws
```
</Terminal>

### CI/CD Integration

<Terminal title="ci/cd examples">
```bash
# Security gate in pipeline
atmos ai security --stack prod-us-east-1 --severity critical --format json > security.json
if jq -e '.findings | length > 0' security.json; then
  echo "Critical security findings detected"
  exit 1
fi

# Generate security report as a deployment artifact
atmos ai security --stack prod-us-east-1 \
  --severity critical,high \
  --format markdown > security-report.md
```
</Terminal>

## Configuration

Configure the security command in your `atmos.yaml` under the `ai.security` section:

```yaml
ai:
  security:
    default_severity:
      - critical
      - high
    sources:
      - security-hub
      - inspector
      - guardduty
    max_findings: 100
```
