:::note Purpose
Use this command to show detailed information about Atmos stacks with rich output formatting options.
:::

<Screengrab title="atmos describe stacks --help" slug="atmos-describe-stacks--help" />

## Usage

Execute the `describe stacks` command like this:

```shell
# Show detailed information for all stacks
atmos describe stacks

# Show specific fields in JSON format
atmos describe stacks --json name,components

# Transform JSON output using JQ
atmos describe stacks --json name,components --jq '<query>'

# Format output using Go templates
atmos describe stacks --json name,components --template '<template>'
```

:::tip
Run `atmos describe stacks --help` to see all the available options
:::

## Examples

```shell
# Show available JSON fields
atmos describe stacks --json

# Show specific stack fields
atmos describe stacks --json name,components,vars

# Transform output using JQ queries
atmos describe stacks --json name,components --jq 'to_entries | map({
  stack: .key,
  components: .value.components.terraform | keys
})'

# Format output using Go templates
atmos describe stacks --json name,components --template '{{range $stack, $data := .}}{{tablerow $stack ($data.components.terraform | keys | join ", ")}}{{end}}'
```

## Flags

| Flag             | Description                                                                                                                                                                                                                               | Required |
|------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|----------|
| `--json`         | Comma-separated list of fields to include in JSON output (use without value to see available fields)                                                                                                                                     | no       |
| `--jq`          | JQ query to transform JSON output (requires --json)                                                                                                                                                                                     | no       |
| `--template`     | Go template to format JSON output (requires --json)                                                                                                                                                                                     | no       |

## Output Formatting

### JSON Field Selection
Use `--json` to specify which fields to include in the output:
```shell
# Show available fields
atmos describe stacks --json

# Select specific fields
atmos describe stacks --json name,components,vars
```

### JQ Queries
Use `--jq` for JSON transformation (requires `--json`):
```shell
# List stack names only
atmos describe stacks --json name --jq 'keys'

# Create custom structure
atmos describe stacks --json name,components --jq 'to_entries | map({
  stack: .key,
  components: .value.components.terraform | keys
})'
```

### Go Templates
Use `--template` for custom formatting (requires `--json`). Supports:
- Go template syntax
- Sprig functions
- Custom functions:
  - `autocolor`: Automatic terminal coloring
  - `color`: Explicit color styling (success, error, warning, info)
  - `tablerow`: Table-style output formatting

```shell
# Simple table format
atmos describe stacks --json name,components --template '{{range $stack, $data := .}}{{tablerow $stack ($data.components.terraform | keys | join ", ")}}{{end}}'

# Colored output
atmos describe stacks --json name,components --template '{{range $stack, $data := .}}{{tablerow (color "success" $stack) (autocolor ($data.components.terraform | keys | join ", "))}}{{end}}'
```

:::tip
The template functionality supports all standard Go template functions plus the complete set of [Sprig functions](http://masterminds.github.io/sprig/).
:::
