<Intro>
Display detailed metadata and information about a stored planfile without downloading it. Useful for verifying planfile contents before applying.
</Intro>

## Usage

```shell
atmos terraform planfile show <key> [options]
```

## Arguments

<dl>
  <dt>`<key>`</dt>
  <dd>Storage key of the planfile to show (required). Format: `{stack}/{component}/{sha}.tfplan`</dd>
</dl>

## Flags

<dl>
  <dt>`--store`</dt>
  <dd>Storage backend to use: `local`, `s3`, or `github-artifacts`. Auto-detected from environment.</dd>

  <dt>`--format`</dt>
  <dd>Output format: `text` (default) or `json`.</dd>
</dl>

## Examples

### Show planfile details

```shell
atmos terraform planfile show dev-us-east-1/vpc/abc123.tfplan
```

### JSON output

```shell
atmos terraform planfile show dev-us-east-1/vpc/abc123.tfplan --format json
```

### Show from specific store

```shell
atmos terraform planfile show dev-us-east-1/vpc/abc123.tfplan --store s3
```

## Use Cases

### Verify before apply

Check planfile contents before applying in a separate CI job:

```shell
# Verify the planfile exists and has expected changes
atmos terraform planfile show "$STACK/$COMPONENT/$SHA.tfplan"

# Then apply
atmos terraform apply $COMPONENT -s $STACK --from-plan=$PLANFILE
```

### Audit planfile history

Review what changes were planned:

```shell
atmos terraform planfile list staging | head -5
atmos terraform planfile show staging/rds/latest.tfplan
```
