<Intro>
Use this command to display the `metadata.source` configuration for a Terraform component. This shows the source URI, version, and any path filters configured for vendoring.
</Intro>

<Screengrab title="atmos terraform source describe --help" slug="atmos-terraform-source-describe--help" />

## Usage

```shell
atmos terraform source describe <component> --stack <stack>
```

## Description

The `describe` command extracts and displays the `metadata.source` configuration from a component's stack manifest. This is useful for:

- Verifying source configuration before vendoring
- Checking the version configured for a component
- Reviewing included/excluded path filters
- Debugging source provisioning issues

## Examples

### View Source Configuration

Display the source configuration for a component:

```shell
atmos terraform source describe vpc --stack dev
```

Output:
```yaml
component: vpc
stack: dev
source:
  uri: github.com/cloudposse/terraform-aws-components//modules/vpc
  version: 1.450.0
  included_paths:
    - "*.tf"
    - "modules/**"
  excluded_paths:
    - "*.md"
    - "tests/**"
```

### String Format Source

When source is configured as a simple string:

```shell
atmos terraform source describe eks --stack dev
```

Output:
```yaml
component: eks
stack: dev
source:
  uri: github.com/cloudposse/terraform-aws-components//modules/eks?ref=1.450.0
  version: ""
```

## Arguments

<dl>
  <dt>`component`</dt>
  <dd>**Required.** The name of the Atmos component to describe.</dd>
</dl>

## Flags

<dl>
  <dt>`--stack` / `-s`</dt>
  <dd>**Required.** The Atmos stack name. Can also be set via `ATMOS_STACK` environment variable.</dd>
</dl>

## Output Format

The command outputs YAML with the following structure:

<dl>
  <dt>`component`</dt>
  <dd>The component name provided as input.</dd>

  <dt>`stack`</dt>
  <dd>The stack name provided as input.</dd>

  <dt>`source`</dt>
  <dd>The parsed source specification.</dd>

  <dt>`source.uri`</dt>
  <dd>The go-getter compatible source URI.</dd>

  <dt>`source.version`</dt>
  <dd>The version tag, branch, or commit (if specified separately).</dd>

  <dt>`source.included_paths`</dt>
  <dd>List of glob patterns for files to include.</dd>

  <dt>`source.excluded_paths`</dt>
  <dd>List of glob patterns for files to exclude.</dd>
</dl>
