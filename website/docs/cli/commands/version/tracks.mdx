<Intro>
Use `atmos version tracks` to inspect, lock, update, verify, and render Atmos-managed external versions.
</Intro>

Version tracks are configured in [`version.tracks`](/cli/configuration/version/managed-versions) in `atmos.yaml`. Resolved versions are stored in `versions.lock.yaml` by default.

## Usage

```shell
atmos version tracks <command>
```

## Commands

| Command | Description |
|---------|-------------|
| `list` | List configured tracks. |
| `show <track>` | Show the effective entries for a track after defaults and groups are applied. |
| `lock <track>` | Resolve and write lock entries for a track. |
| `update <track>` | Update lock entries for a track. |
| `status <track>` | Show locked, resolved, and update status. |
| `diff <track>` | Show entries that are unlocked or differ from the resolved target. |
| `verify <track>` | Verify that a track is locked and current. |
| `render <track>` | Render a file containing `{{ .version.name }}` references. |

## Examples

List tracks:

```shell
atmos version tracks list
```

Show the effective production track:

```shell
atmos version tracks show prod
```

Lock all versions in a track:

```shell
atmos version tracks lock prod
```

Update only the `infrastructure` group:

```shell
atmos version tracks update prod --group infrastructure
```

Check status as JSON for CI:

```shell
atmos version tracks status prod --format json
```

Verify that the lock file is current:

```shell
atmos version tracks verify prod
```

Render a GitHub Actions workflow template:

```shell
atmos version tracks render prod \
  --file .github/workflows/deploy.yaml.tmpl \
  --output .github/workflows/deploy.yaml
```

Check that a rendered file is up to date:

```shell
atmos version tracks render prod \
  --file .github/workflows/deploy.yaml.tmpl \
  --output .github/workflows/deploy.yaml \
  --check
```

## Flags

<dl>
  <dt>`--format`</dt>
  <dd>Output format: `yaml`, `json`, or `table`. The default is `yaml`.</dd>

  <dt>`--group`</dt>
  <dd>Limit `lock`, `update`, `status`, or `diff` to one configured version group.</dd>

  <dt>`--file`</dt>
  <dd>Source template file for `render`.</dd>

  <dt>`--output`</dt>
  <dd>Destination file for `render`. If omitted, rendered output is written to stdout.</dd>

  <dt>`--check`</dt>
  <dd>For `render`, fail if the destination file does not match rendered output.</dd>
</dl>

## CI Usage

Use `status`, `verify`, and `render --check` in CI to make version drift visible before applying infrastructure changes:

```yaml
jobs:
  version-check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v6
      - run: atmos version tracks status prod --format json
      - run: atmos version tracks verify prod
      - run: atmos version tracks render prod --file .github/workflows/deploy.yaml.tmpl --output .github/workflows/deploy.yaml --check
```

GitHub Actions workflow files are parsed by GitHub before Atmos runs, so author workflow templates with `{{ .version.name }}` and commit the rendered workflow YAML with literal refs.
