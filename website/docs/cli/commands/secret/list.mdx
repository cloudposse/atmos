<Intro>
List the declared secrets for a stack and component as a table showing STACK, COMPONENT, SECRET, PROVIDER, and STATUS. The status reflects a backend existence check (`initialized`, `missing`, or `error`), so `list` never retrieves secret values and never registers them for masking.
</Intro>

<Screengrab title="atmos secret list --help" slug="atmos-secret-list--help" />

## Usage

```shell
atmos secret list [flags]
```

## Examples

```shell
# List declared secrets and their status
atmos secret list --stack=prod --component=api

# Include declaration descriptions in the output
atmos secret list --stack=prod --component=api --verbose

# Disambiguate a component that exists in multiple types
atmos secret list --stack=prod --component=api --type=terraform

# List using a specific identity for the backend status check
atmos secret list --stack=prod --component=api --identity=aws/prod-secrets
```

## Arguments

<dl>
  <dt>n/a</dt>
  <dd>No positional arguments.</dd>
</dl>

## Flags

<dl>
  <dt>`--stack` <em>(alias `-s`)</em></dt>
  <dd>
    The Atmos stack to operate on. **Required.**

    **Environment variable:** `ATMOS_STACK`
  </dd>

  <dt>`--component` <em>(alias `-c`)</em></dt>
  <dd>
    The Atmos component whose declared secrets are listed. **Required.**

    **Environment variable:** `ATMOS_COMPONENT`
  </dd>

  <dt>`--type`</dt>
  <dd>
    The component type (`terraform`, `helmfile`, `packer`, or `ansible`). Used to disambiguate when a component name exists in more than one type.
  </dd>

  <dt>`--identity` <em>(alias `-i`)</em></dt>
  <dd>
    The identity to use when accessing the secret backend.

    **Environment variable:** `ATMOS_IDENTITY`
  </dd>

  <dt>`--verbose` <em>(alias `-v`)</em></dt>
  <dd>
    Include each secret's declaration description in the output.
  </dd>
</dl>
