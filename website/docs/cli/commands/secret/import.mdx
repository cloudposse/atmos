<Intro>
Import secret values from a `.env` or JSON file into the configured backend. Declared keys are set, while undeclared keys are warned about and **skipped** rather than causing a failure. When the import completes, Atmos reports a summary of how many values were imported and how many were skipped. Use [`push`](/cli/commands/secret/push) instead if undeclared keys should be a hard error.
</Intro>

<Screengrab title="atmos secret import --help" slug="atmos-secret-import--help" />

## Usage

```shell
atmos secret import FILE [flags]
```

## Examples

```shell
# Import values from a .env file
atmos secret import secrets.env --stack=prod --component=api

# Import values from a JSON file
atmos secret import secrets.json --format=json --stack=prod --component=api

# Preview the import without writing anything
atmos secret import secrets.env --dry-run --stack=prod --component=api

# Import values piped in from standard input
cat secrets.env | atmos secret import - --stack=prod --component=api
```

## Arguments

<dl>
  <dt>`FILE`</dt>
  <dd>
    The path to the file to import secret values from. Use `-` to read from standard input.
  </dd>
</dl>

## Flags

<dl>
  <dt>`--stack` <em>(alias `-s`)</em></dt>
  <dd>
    The Atmos stack to operate on. **Required.**

    **Environment variable:** `ATMOS_STACK`
  </dd>

  <dt>`--component` <em>(alias `-c`)</em></dt>
  <dd>
    The Atmos component whose declared secrets receive the values. **Required.**

    **Environment variable:** `ATMOS_COMPONENT`
  </dd>

  <dt>`--type`</dt>
  <dd>
    The component type (`terraform`, `helmfile`, `packer`, or `ansible`). Used to disambiguate when a component name exists in more than one type.
  </dd>

  <dt>`--identity` <em>(alias `-i`)</em></dt>
  <dd>
    The identity to use when accessing the secret backend.

    **Environment variable:** `ATMOS_IDENTITY`
  </dd>

  <dt>`--format`</dt>
  <dd>
    The input format: `env` or `json`. Defaults to `env`.
  </dd>

  <dt>`--dry-run`</dt>
  <dd>
    Preview which values would be imported and which would be skipped, without writing anything to the backend.
  </dd>
</dl>

:::note Lenient by design
Unlike [`push`](/cli/commands/secret/push), `import` does not fail on undeclared keys. It warns about each undeclared key, skips it, and reports a summary (for example, `5 imported, 2 skipped`).
:::
