<Intro>
Use this command to add an external MCP server to the `mcp.servers` section of `atmos.yaml`.
</Intro>

<Experimental />

## Description

The `atmos mcp add` command adds a new MCP server configuration to your `atmos.yaml` file. The server configuration follows the standard MCP format used by Claude Code, Codex CLI, and Gemini CLI.

## Usage

```shell
atmos mcp add <name> [flags]
```

### Arguments

<dl>
  <dt>`name`</dt>
  <dd>The name for the MCP server (used in other commands like `atmos mcp test <name>`).</dd>
</dl>

### Flags

<dl>
  <dt>`--command`, `-c`</dt>
  <dd>Command to run the MCP server (e.g., `uvx`, `npx`). Required.</dd>

  <dt>`--args`, `-a`</dt>
  <dd>Arguments for the server command. Can be specified multiple times.</dd>

  <dt>`--description`, `-d`</dt>
  <dd>Human-readable description shown in `atmos mcp list`.</dd>

  <dt>`--env`, `-e`</dt>
  <dd>Environment variables in `KEY=VALUE` format. Can be specified multiple times.</dd>
</dl>

### Examples

```shell
# Add an AWS documentation server
atmos mcp add aws-docs \
  --command uvx \
  --args "awslabs.aws-documentation-mcp-server@latest" \
  --description "AWS Documentation"

# Add a server with environment variables
atmos mcp add aws-pricing \
  --command uvx \
  --args "awslabs.aws-pricing-mcp-server@latest" \
  --env AWS_REGION=us-east-1 \
  --env FASTMCP_LOG_LEVEL=ERROR

# Verify the server was added
atmos mcp list
```
