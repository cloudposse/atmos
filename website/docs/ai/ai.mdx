## Quick Start

<File title="atmos.yaml">
```yaml
ai:
  enabled: true
  default_provider: "anthropic"
  providers:
    anthropic:
      model: "claude-sonnet-4-6"
      api_key: !env "ANTHROPIC_API_KEY"
```
</File>

<Terminal title="shell">
```bash
export ANTHROPIC_API_KEY="your-api-key"
atmos ai chat                                        # Interactive chat
atmos ai ask "What stacks do we have?"               # Single question
atmos ai exec "validate stacks" --format json        # CI/CD automation
atmos ai sessions list                               # List sessions
atmos ai skill list                                  # List installed skills
```
</Terminal>

## Commands

<dl>
  <dt>[`atmos ai chat`](/cli/commands/ai/chat)</dt>
  <dd>Interactive chat with session management, provider switching, and skill selection.</dd>

  <dt>[`atmos ai ask`](/cli/commands/ai/ask)</dt>
  <dd>Ask a single question and get an immediate response. Ideal for scripting and CI/CD.</dd>

  <dt>[`atmos ai exec`](/cli/commands/ai/exec)</dt>
  <dd>Execute Atmos and shell commands via AI prompts with structured output.</dd>

  <dt>[`atmos ai sessions`](/cli/commands/ai/sessions)</dt>
  <dd>Manage chat sessions: list, clean, export, and import.</dd>

  <dt>[`atmos ai skill`](/cli/commands/ai/skill)</dt>
  <dd>Install, list, and uninstall community AI skills from GitHub.</dd>

  <dt>[`atmos mcp start`](/cli/commands/mcp/start)</dt>
  <dd>Start the MCP server with stdio (default) or HTTP transport.</dd>

  <dt>[`atmos lsp start`](/cli/commands/lsp/start)</dt>
  <dd>Start the LSP server with stdio, TCP, or WebSocket transport.</dd>
</dl>

## MCP Server

The [Model Context Protocol](https://modelcontextprotocol.io) (MCP) server exposes Atmos AI tools
to any compatible client -- Claude Desktop, Claude Code, VS Code, Cursor, Gemini CLI, and
[many others](https://modelcontextprotocol.io/clients). This lets you use Atmos tools inside your
preferred AI assistant without custom integrations.
