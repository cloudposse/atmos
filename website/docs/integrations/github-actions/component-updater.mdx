<Intro>
Atmos’s native Component Updater keeps vendored components current without hand-editing version
pins. It discovers allowed upstream releases, shows exactly what changed, updates the declaring
manifest without losing its formatting, and can pull the updated component sources.
</Intro>

The Component Updater replaces the version-discovery and manifest-editing work previously handled
by the legacy GitHub Action. Run it locally or directly from CI with the same `atmos` commands.

:::note Legacy GitHub Action

`cloudposse/github-action-atmos-component-updater` is deprecated. Its documentation remains
available as a [legacy reference](/deprecated/github-actions/component-updater).

:::

## Review available updates

Start with a dry run. Atmos checks each Git-backed source against its version constraints and
reports the updates it would make without changing files:

```shell
atmos vendor update --check
```

Use `atmos vendor diff` to inspect the source change before accepting an update:

```shell
# Compare the pinned version with the latest tag.
atmos vendor diff --component vpc

# Compare two explicit versions, optionally limited to one file.
atmos vendor diff --component vpc --from 1.0.0 --to 2.0.0 --diff-file variables.tf
```

## Update components

Update all eligible Git-backed sources, one component, or a tag-selected set:

```shell
# Update every eligible source in the manifest.
atmos vendor update

# Update one component or all components carrying a tag.
atmos vendor update --component vpc
atmos vendor update --tags networking

# Update versions, then pull the updated component sources.
atmos vendor update --pull
```

Atmos writes the `version` field in place while preserving comments, YAML anchors, and Go
templates. It follows the same precedence as `atmos vendor pull`: a repository-level `vendor.yaml`
takes precedence, with support for per-component `component.yaml` manifests when no matching
vendor source exists.

## Version policy

Each source can constrain the versions Atmos may select, including semantic-version ranges,
excluded releases, and prerelease handling. See the
[`atmos vendor update` reference](/cli/commands/vendor/vendor-update#version-constraints) for
the full configuration and flag reference.
