# Implementation Plan: Pact Consumer Contract Testing for Atmos Pro API

**Branch**: `1198-pact-consumer-contracts` | **Date**: 2026-06-09 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `specs/001-pact-consumer-contracts/spec.md`

## Summary

Add pact consumer contract tests for all 8 Atmos Pro API endpoints in `pkg/pro/`. Each
test runs the real `AtmosProAPIClient` code against a pact mock HTTP server, generating
JSON contract files in `pacts/` that describe the exact request/response shapes the
Atmos client expects. Tests are isolated behind a `//go:build pact` tag so they never
slow the standard `go test ./...` run. No Pact Broker, no CI job — local-only developer
tooling. The README gains a dedicated "Pact Contract Testing" section.

## Technical Context

**Language/Version**: Go 1.26.3

**Primary Dependencies**:
- `github.com/pact-foundation/pact-go/v2` (new dev dependency — pact mock server + DSL)
- `github.com/stretchr/testify` (existing — assertions)
- `pkg/pro` (existing — `AtmosProAPIClient` under test)

**Storage**: Local `pacts/` directory at repository root — generated JSON pact files.
These are checked into version control so contract drift is visible in PR diffs.

**Testing**: `go test -tags pact ./pkg/pro/...`

**Target Platform**: Linux, macOS, Windows (pact-go v2 ships pre-built native libs for
all three via `pact-go install`).

**Project Type**: Go test infrastructure (no new CLI commands, no new packages beyond
test files).

**Performance Goals**: N/A — tests run offline against a loopback mock server;
sub-second per interaction is expected.

**Constraints**:
- Build tag `pact` isolates tests from standard suite.
- No Pact Broker publishing.
- Pact tests are local-only; no dedicated CI job.
- pact-go v2 native libs must be installed separately via `pact-go install`.

**Scale/Scope**: 8 consumer interactions across 1 consumer–provider pair
(`atmos` → `AtmosPro`).
