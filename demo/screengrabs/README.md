# Screengrabs

Documentation screengrabs are asciicast recordings (`.cast`), generated by a Go
driver (`main.go`/`grabber.go`) built on the Atmos asciicast tooling
(`pkg/asciicast`). Each command in the manifest is recorded, noise-filtered,
and committed as a single `.cast` file. The website renders them client-side
with the `CastPlayer` component in static mode — a single frame of the full
output — so no HTML, image, or ANSI-conversion artifacts are generated or
committed. No external tools (`aha`, `script`, sed hacks) are required.

Story-style demo casts (e.g. the `learn/mindset` walkthroughs) are not part of
this manifest; they are Atmos custom commands under
`demo/casts/atmos.d/screengrabs/` (run `atmos casts generate screengrabs
demo-stacks` from `demo/casts`).

## Usage

Add all commands to `demo-stacks.txt` (use `#` to disable generation of the command).

Recording runs natively on any OS — no containers or Homebrew required. Casts
are captured without a PTY; Atmos produces correct TrueColor output and layout
width from the `ATMOS_FORCE_COLOR` and `COLUMNS` environment variables the
grabber sets. Recording dependencies (`bat`, `tree`, `terraform`) are installed
via the Atmos toolchain, and the `atmos` binary is built from the working tree.

Build all casts and install them into `../../website/static/casts/screengrabs`
by running:

```shell
make all
```

(Or run the steps separately: `make deps`, `make build-all`, `make install`.)

All the files in `website/static/casts/screengrabs` should be committed.

Embed a screengrab inside any MDX documentation file like this:

```mdx
import CastPlayer from '@site/src/components/CastPlayer'

<CastPlayer title="atmos help" src="/casts/screengrabs/atmos--help.cast" static chrome />
```
