# Screengrabs

Documentation screengrabs are generated by a Go driver (`main.go`/`grabber.go`)
built on the Atmos asciicast tooling (`pkg/asciicast`). Each command in the
manifest is recorded as an asciicast and rendered natively to:

- `<slug>.html` — inline-styled HTML fragment used by the website `Screengrab` component
- `<slug>.ascii` — plain-text output (no ANSI codes), a cheap, diffable artifact readable by other tooling (e.g. Atmos Pro)

No external ANSI-to-HTML converter (`aha`) or `script`/sed hacks are required.

## Usage

Add all commands to `demo-stacks.txt` (use `#` to disable generation of the command).

Build all artifacts by running:

```shell
make build-all
```

Then install them into `../../website/src/components/Screengrabs` by running:

```shell
make install
```

All the files in `website/src/components/Screengrabs` should be committed.

On macOS (or if dependencies are missing), generate with a Linux container instead:

```shell
make docker-all
```

Use these screengrabs inside of any MDX documentation file like this:

```mdx
import Screengrab from '@site/src/components/Screengrab'

<Screengrab title="atmos help" slug="atmos--help" />
```

> [!TIP]
> If you have multiple screengrabs inside the same `.mdx` file, make sure to name each component something unique.
